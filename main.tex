\section{Introduction and Motivation}
\subsection{Context: Biological Interpretation as Mechanistic Discovery}
\begin{itemize}
    \item Biological interpretation involves the development/discovery of mechanistic relationships between biological entities such as genes, proteins, RNA, and other molecular components 

    \item It is essential for discovering context-specific relationships and developing models of disease, phenotypes, pathways, and regulatory systems

    \item In recent years, ML/AI-based methods such as iRF, MENTOR, and related approaches have accelerated biological interpretation through the construction of biologically informed networks, which scientists explore to generate mechanistic and functional hypotheses.

    \item As these methods increase the resolution and scope of inferred networks, biological interpretation has become increasingly mediated by large, densely connected, multi-scale graph structures.
\end{itemize}

\subsection{Problem: Biological Interpretation is Bottlenecked by Human Cognitive Scale}
\begin{itemize}
    \item Although ML/AI methods have refined mechanistic focus into groups of functionally related clades of genes, these clades can still be massive and intractable for individual researchers to systematically explore.

    \item Additionally, most research emphasizes a deep-but-narrow focus on a small subset of functionally related genes, which can overlook broader yet biologically meaningful network context, particularly in highly interactive systems such as living organisms

    \item As biological networks grow in size and complexity, the combinatorial space of possible multi-hop interactions expands rapidly, while human interpretive capacity remains fixed.

    \item This mismatch creates a scaling limit in biological mechanistic interpretation: although computational methods can generate increasingly rich relational structures, the downstream synthesis of mechanistic hypotheses remains constrained by human bandwidth, attention, and cognitive integration capacity.

    \item consequently, the speed, breadth, and contextual depth of mechanistic discovery are limited, particularly for complex traits, polygenic diseases, and multi-layer regulatory systems.
\end{itemize}

\subsection{Resolution: AI-Guided Reasoning to Extend Mechanistic Interpretation Beyond Human Scale}
\begin{itemize}
    \item AI models, particularly LLMs, present an opportunity to aid scientists in mechanistic discovery tasks

    \item LLMs excel at processing and synthesizing large volumes of information, complementing the deep domain expertise and focused inquiry of human researchers

    \item recent developments in agent-based systems enable models to invoke deterministic tools, allowing structured interaction with network-based representations and substantially reducing hallucination

    \item we propose \textbf{MENTOR-RL} a reasoning-based model for structured network exploration and mechanistic discovery designed explicity to address the scaling limits of biological interpretation

    \item by training transformer-based agents with reinforcement learning across a diverse set of tool-use tasks and contextual levels of biological network data, we aim to produce a model that can collaborate with researchers to traverse complex networks, synthesize \textit{multi-hop} mechanistic hypotheses, and integrate both local functional structure and global topological context

    \item Rather than replacing scientific judgment, MENTOR-RL is intended to extend the effective interpretive bandwidth of researchers, enabling broader, faster, and more context-aware mechanistic discovery.

    
\end{itemize}

\section{Proposed Methodology}
\subsection{Agent Loop Formulation}
\begin{itemize}
    \item Formalize the graph exploration task as an iterative agent loop, with query/hypothesis generation being defined at discrete decision points

    \item Working model updates
    \begin{enumerate}
        \item propose a hypothesis
        \item select a graph query (or queries)
        \item interpret returned results
        \item update working model
    \end{enumerate}

    \item each decision is an evaluable action within a sequential decision-making and refinement process
\end{itemize}

\subsection{Training Framework}
\begin{itemize}
    \item \textbf{DO NOT USE FRONTIER MODELS}
    \item Utilize high-quality expert rollouts to construct structured preference datasets over intermediate reasoning steps 

    \item training pipeline:
    \begin{enumerate}
        \item collect rollout traces from a frontier-model based agent interacting with the graph environment through a combination of SQL database queries and NetworkX (or PyG) graph exploration functions
        
        \item extract intermediate decision points

        \item condition a smaller open-source model on the trace context

        \item sample multiple candidate next-step actions

        \item use an expert judge (GPT-5 or other model) to rank candidates

        \item construct preference pairs

        \item optimize with DPO
    \end{enumerate}

    \item DPO operates off-policy, so it doesn't need to produce its own trace. Follows a teacher-student paradigm
\end{itemize}

\subsection{Curriculum and Optimization Strategy}
\begin{itemize}
    \item we need to mitigate the distribution mismatch between expert-generated context and student-generated context during deployment. We will do so with a curriculum to incorporate student-generated context

    \item curriculum: scale from "gold-standard gene sets" to more novel biology
    
    
    \item we can supplement the curriculum with human-edited context to push the model more toward the "expert distribution" as we begin using this in production. Can use either PPO or DPO for this

    \item progessively reduces the covariate shift between training and deployment context distributions 
\end{itemize}

\subsection{Reward Decomposition}
\begin{itemize}
    \item we divide the reward structure between the query (partially verifiable), the hypothesis (subjective), and interpretation (soft judgement)

    \item query: syntactic validity, execution success

    \item hypothesis formation: expert-model judgement

    \item interpretation: expert-model judgement
\end{itemize}

\subsection{Verbalized Sampling and Diversity}
\begin{itemize}
    \item generate multiple, varied answers, alongside explicit probability scores to discourage producing "safe" answers

    \item can be incorporated simply with the rest of the RL pipeline 
\end{itemize}
