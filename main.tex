\section{Introduction and Motivation}
\subsection{Context: Biological Interpretation as Mechanistic Discovery}
\begin{itemize}
    \item Biological interpretation involves the development/discovery of mechanistic relationships between biological entities such as genes, proteins, RNA, and other molecular components 

    \item It is essential for discovering context-specific relationships and developing models of disease, phenotypes, pathways, and regulatory systems

    \item In recent years, ML/AI-based methods such as iRF, MENTOR, and related approaches have accelerated biological interpretation through the construction of biologically informed networks, which scientists explore to generate mechanistic and functional hypotheses.

    \item As these methods increase the resolution and scope of inferred networks, biological interpretation has become increasingly mediated by large, densely connected, multi-scale graph structures.

    \item As biological networks and supporting databases grow, mechanistic exploration requires many iterative queries and multi-step reasoning.
\end{itemize}

\subsection{Problem: Biological Interpretation is Bottlenecked by Scale, both in Human Cognition and Frontier Model Cost}
\begin{itemize}
    \item Although ML/AI methods have refined mechanistic focus into groups of functionally related clades of genes, these clades can still be massive and intractable for individual researchers to systematically explore.

    \item Additionally, most research emphasizes a deep-but-narrow focus on a small subset of functionally related genes, which can overlook broader yet biologically meaningful network context, particularly in highly interactive systems such as living organisms

    \item As biological networks grow in size and complexity, the combinatorial space of possible multi-hop interactions expands rapidly, while human interpretive capacity remains fixed.

    \item While Large Language Models (LLMs) excel at processing and synthesizing large volumes of information, frontier inference cost scales with exploration length and reasoning depth, limiting agentic deployment scenarios.

    \item This mismatch creates a scaling limit in biological mechanistic interpretation: although computational methods can generate increasingly rich relational structures, the downstream synthesis of mechanistic hypotheses remains constrained by human bandwidth, and large language model cost.

    \item consequently, the speed, breadth, and contextual depth of mechanistic discovery are limited, particularly for complex traits, polygenic diseases, and multi-layer regulatory systems.
\end{itemize}

\subsection{Resolution: AI-Guided Reasoning to Extend Mechanistic Interpretation Beyond Human Scale}
\begin{itemize}
    \item Open-source LLMs present an opportunity to aid scientists in mechanistic discovery tasks without being cost prohibitive.

    \item recent developments in agent-based systems enable models to invoke deterministic tools, allowing structured interaction with network-based representations and substantially reducing hallucination. Further more, open-source releases allow users to train models to excel at specific tasks while being free to use, besides computational cost.
    
    \item we propose \textbf{MENTOR-RL}, a reasoning agent trained on human-annotated biological ground truth and verifiable interaction objective, enabling scalable mechanistic exploration without a reliance on proprietary models.

    \item by training a transformer-based agent with reinforcement learning across a diverse set of tool-use tasks and contextual levels of biological network data, we aim to produce a model that can collaborate with researchers to traverse complex networks, synthesize \textit{multi-hop} mechanistic hypotheses, and integrate both local functional structure and global topological context.

    \item Leveraging open-source model offerings allows us to deliver a model that can be deployed locally, with training cost amortized across many exploration sessions.

    \item Rather than replacing scientific judgment, MENTOR-RL is intended to extend the effective interpretive bandwidth of researchers, enabling broader, faster, and more context-aware mechanistic discovery.

    
\end{itemize}

\section{Proposed Methodology}
\subsection{Agent Loop Formulation}
\begin{itemize}
    \item Formalize the graph exploration task as an iterative agent loop, with query/hypothesis generation being defined at discrete decision points

    \item Working model updates
    \begin{enumerate}
        \item propose a hypothesis
        \item select a graph query (or queries)
        \item interpret returned results
        \item update working model
    \end{enumerate}

    \item model structured state: alongside the working hypothesis, used for verifiable rewards
    \begin{enumerate}
        \item current seed set
        \item candidate protein set
        \item evidence edges / provenance
        \item predicted complex id
        \item stop / continue decision
    \end{enumerate}

    \item Model actions are emitted in a machine-parsable schema (JSON):
    \begin{itemize}
        \item tool call
        \item proposed state update
        \item mechanistic labels
        \item short finding statement
        \item stop/continue
    \end{itemize}

    \item each decision is an evaluable action within a sequential decision-making and refinement process
\end{itemize}

\subsection{Task Outputs and Mechanistic Targets}
\begin{itemize}
    \item The agent optimizes three outputs:
    \begin{enumerate}
        \item Complex membership prediction $\hat{C}$
        
        \item Mechanistic labels $\hat{y}_{\text{mech}}$: localization, functional category, and signature terms
        
        \item Evidence-grounded mechanistic summary
    \end{enumerate}
    
    \item Mechanistic ground truth $y_{\text{text}}$ is derived from CORUM curated annotations: function, localization, and signature terms
\end{itemize}

\subsection{Training Framework}
\begin{itemize}

    \item Tool call/ format SFT for proper tool calling

    \item Agent-Derived SFT for gene info/world-knowledge

    \item evidence $\rightarrow$ interpretation SFT
    \begin{itemize}
        \item input: tool observations + structured state
        \item target: mechanistic label prediction, short mechanistic finding/summary
    \end{itemize}
    
    \item implement two RL training modes

    \item 1: preference optimization with CORUM labels
    \begin{itemize}
        \item context $x$: full trajectory
        \item candidates ${y_i}$: next-step proposals $\rightarrow$ query, update, and hypothesis
        \item score each candidate via deterministic metrics \ref{sec:rewards}
        \item construct preference pairs: best vs. worst output or pairwise ranking, producing a dataset of $\{prompt, chosen, rejected\}$.
        \item TRL DPOTrainer supports tool-calling datasets with tool calls, tool responses, and a tools call with schemas
        \item optimize with DPO 
    \end{itemize}

    \item 1.1: DPO training loop
    \begin{itemize}
        \item at decision point $t$, we have target complex proteins $C$, current structured state $s_t$, and current prompt/working model $x_t$

        \item sample $N$ candidates from the base model/SFT checkpoint: $$ y_1, ..., y_N = \text{sample}(\pi_{\theta}(\cdot | x_t))$$

        \item score tool validity and formatting:
        \begin{itemize}
            \item $$\text{tool}(y_i) \in \{0,1\}$$
            \item $$\text{format}(y_i) \in \{0, 1\}$$
        \end{itemize}

        \item let candidate propose updated prediction protein set $$\hat{C}_{t+1}$$

        \item compute complex/exploration reward:
        \begin{itemize}
            \item $\Delta F1 = F1(C, \hat{C}_{t+1}) - F1(C, \hat{C}_{t}) $
            \item or Jaccard, Recall
        \end{itemize}

        \item combine rewards
    \end{itemize}
    
    

    \item 2: On-policy RL (PPO/GRPO) once rollouts are coherent
    \begin{itemize}
        \item full episodes with model controlling actions
        \item optimize policy directly
    \end{itemize}

    \item DPO improves model coherence in early stages
    \item PPO/GRPO expand to long horizons

\end{itemize}

\subsection{Reward Design and Decomposition}\label{sec:rewards}
\begin{itemize}
    \item query validity, tool discipline (dense + cheap rewards)
    \begin{itemize}
        \item positive: tool call is syntactically valid, executes
        \item positive: returned results are non-empty, properly formatted
        \item negative: per tool call to encourage efficiency
        \item negative: penalty for repeated invalid queries/loops
    \end{itemize}

    \item complex recovery reward (mechanistic ground truth)
    \begin{itemize}
        \item define $C$ as the ground-truth corum complex, $\hat{C}$ as the model-predicted complex
        \item use set-based metrics: F1, jaccard overlap, precision@k, recall@k
        \item dense shaping: positive reward when model adds a correct, unseen member of a complex, penalize when an incorrect protein is added
    \end{itemize}

    \item topological/contextual rewards
    \begin{itemize}
        \item positively reward coverage of multiple neighborhooods
        \item penalize revisits / redundant expansion
    \end{itemize}

    \item mechanistic interpretation reward (ground truth, groundedness)
    \begin{itemize}
        \item compare predicted mechanistic labels to CORUM annotations
        \item reward evidence-grounded summaries, and penalize unsupported claims (+ for mentioning retrieved terms, - for mentioning unretrieved terms)
    \end{itemize}
\end{itemize}

\subsection{Curriculum and Optimization Strategy}
\begin{itemize}
    \item 1: intra-complex tasks (easy, dense rewards)
    \begin{itemize}
        \item complex completion from 1-3 seeds
        \item short horizon, high reward density
    \end{itemize}

    \item 2: harder intra-complex, decoys
    \begin{itemize}
        \item degree-matched random negatives 
        \item partial complexes with missing edges / noisy graphs
    \end{itemize}

    \item 3: inter-complex reasoning
    \begin{itemize}
        \item two complexes connected by shared pathways/functional linkage
        \item requires exploration outside of local topology, encourages RWR usage
        \item can measure correct complex separation
    \end{itemize}
\end{itemize}

\subsection{Verbalized Sampling and Diversity}
\begin{itemize}
    \item use verbalized sampling to produce diverse candidate actions
    \item CORUM-scored ranking to choose preference pairs
    \item prevents mode collapse
\end{itemize}
