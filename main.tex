\section{Introduction and Motivation}

\subsection{Biological Interpretation as Mechanistic Discovery}
Biological interpretation involves the development/discovery of mechanistic relationships between biological entities such as genes, proteins, RNA, and other molecular components. Interpretation is essential for discovering context-specific relationships and developing models of disease, phenotypes, pathways, and regulatory systems. In recent years, machine learning (ML) and artificial intelligence (AI) techniques such as iterative random forest (iRF) \cite{basu2018iterative, cliff2019high}, MENTOR \cite{mentor}, and related approaches have accelerated biological research through the construction of biologically informed networks, which scientists explore to generate mechanistic and functional hypotheses.

As these methods increase the resolution and scope of inferred networks, biological interpretation has become increasingly mediated by large, densely connected, multi-scale graph structures. To comprehensively examine these large biological networks and supporting databases, mechanistic exploration requires iterative querying and multi-step reasoning. Recent agentic interpretation systems \cite{eubiota} demonstrate that provenance-preserving retrieval and tool-mediated synthesis can yield high-fidelity mechanistic narratives, but the approach remains expensive and not fully optimized as a trainable policy.

\subsection{Biological Interpretation is Bottlenecked by Scale, both in Human Cognition and Frontier Model Cost}
Although ML/AI methods have refined mechanistic focus into groups of functionally related clades of genes, these clades can still be massive and intractable for individual researchers to systematically explore. Additionally, most research emphasizes a deep-but-narrow focus on a small subset of functionally related genes, which can overlook broader, yet biologically meaningful, network context, particularly in highly interactive systems such as living organisms. As biological networks grow in size and complexity, the combinatorial space of possible multi-hop interactions expands rapidly, while human interpretive capacity remains fixed.

While Large Language Models (LLMs) can synthesize large volumes of information, frontier inference cost scales with exploration length and reasoning depth, limiting agentic deployment for long-horizon network traversal. This mismatch creates a scaling limit in biological mechanistic interpretation: rich relational structures can be computed, but downstream synthesis remains constrained by human bandwidth and model cost. Consequently, the speed, breadth, and contextual depth of mechanistic discovery are limited, particularly for complex traits, polygenic diseases, and multi-layer regulatory systems.

\subsection{AI-Guided Reasoning to Extend Mechanistic Interpretation Beyond Human Scale}
Open-source LLMs present an opportunity to aid scientists in mechanistic discovery tasks without being cost prohibitive. Recent developments in agent-based systems enable models to invoke deterministic tools with structured interfaces, substantially reducing hallucination and enabling provenance-preserving exploration. We propose \textbf{MENTOR-RL}, a reasoning agent trained on human-annotated biological ground truth and verifiable interaction objectives, enabling scalable mechanistic exploration without reliance on proprietary models.

By training a transformer-based agent with reinforcement learning across tool-use tasks and contextual levels of biological network data, we aim to produce a model that can traverse complex networks, synthesize \textit{multi-hop} mechanistic hypotheses, and integrate both local functional structure and global topological context. Leveraging open-source model offerings allows local deployment, with training cost amortized across many exploration sessions. Rather than replacing scientific judgment, MENTOR-RL extends researcher interpretive bandwidth, enabling broader, faster, and more context-aware mechanistic discovery.

\subsection{Agent Loop Formulation}\label{sec:agent_loop}
We model mechanistic graph exploration as a sequential decision process executed by a single policy model with an internal self-verification step. The agent loop begins with the user providing a query, $q$, and, optionally, evidence $e^{\text{user}}$. Evidence can take the form of additional text context, seed genes, or a multiplex network. These inputs initialize the working interpretation $I_0$ and the structured state $S_0$

For each decision point $t < t^{\text{max}}$, we define the current decision context as

\[
x_t = \{q, e^{\text{user}}, I_t, S_t\},
\]

The agent will emit an action $a_t$, given the query, evidence, current interpretation, and current structured state. This action may be either a text-based planning/reasoning action, a runtime-based tool-calling action, or both. If the action contains a tool-calling step, the tool call will be executed via a deterministic runtime.

 After the action is sampled from the policy and all deterministic tool calls are executed, a verification step takes place. Given the query, evidence, current interpretation, current structured state, and current action/tool outputs, the verifier policy updates the interpretation and structured state. \subsection{Task Outputs and Mechanistic Targets}\label{sec:task}
\begin{itemize}
    \item The agent optimizes three outputs:
    \begin{enumerate}
        \item Complex membership prediction $\hat{C}$
        \item Mechanistic labels $\hat{y}_{\text{mech}}$: localization, functional category, signature terms
        \item Evidence-grounded mechanistic summary (with explicit provenance pointers to retrieved evidence)
    \end{enumerate}

    \item Mechanistic ground truth is derived from CORUM curated annotations (function/localization/signature terms) augmented with tool-retrieved context (GO terms, pathway membership, and literature abstracts).
\end{itemize}

\subsection{Data, Resources, and Environment}\label{sec:data}
Our dataset will be constructed from a combination of existing internal tool infrastructure, open-source data, and graph exploration environments. First, we will construct a tool and database access data subset, relying on existing internal agentic infrastructure and previously developed tools from the MENTOR-IA project. These tools include gene metadata, gene ontology (GO) annotations, and pathway membership information, retrieved via MyGene.info, KEGG, and Reactome. Additionally, we will include tools to access literature abstracts with EMBL-EBI and PubMed ID (PMID) retrieval tools. Using these tools, we will create a dataset of general-knowledge biology tasks, comprising both tool-enabled (open-book) and tool-disabled (closed-book) variants. Examples include retrieving the correct ensembl ID for a given gene name, returning the correct GO annotations for a given gene or gene set, and returning the relevant genes or pathways detailed in an abstract. This dataset will be used for warm-start SFT.

\begin{itemize}

    \item CORUM-grounded training corpus:
    \begin{itemize}
       \item Sample a CORUM complex $C$; given $k$ seed proteins from $C$, recover members and mechanistic labels, or abstain.
       \item Negative examples: distinguish a true complex from degree/size-matched decoys.
       \item Construct per-complex mechanistic targets: $\{C,\text{function\_text},\text{localization},\text{functional\_category},\text{references}\}$.
    \end{itemize}

    \item Graph environment (local multiplex):
    \begin{itemize}
        \item Multiplex graph composed of PPI + co-expression + TF-target + pathway/evidence layers (consistent with prior MENTOR multiplex usage).
        \item Accessed via SQL/graph functions (neighbors, shortest path, random walk, subgraph extraction).
        \item Optional subnetwork induction utilities (e.g., shortest-path-based subgraph extraction).
    \end{itemize}
\end{itemize}

\subsection{Mechanistic Summary Dataset (tool use $\rightarrow$ evidence $\rightarrow$ finding $\rightarrow$ summary)}\label{sec:mechanistic-summary}
\begin{itemize}
    \item Goal: produce training episodes that explicitly connect tool calls and observations to mechanistic conclusions.

    \item For each CORUM complex:
    \begin{enumerate}
        \item initialize with $k$ seeds and a minimal study/background prompt
        \item run an exploration episode (log actions + observations + state updates)
        \item extract a per-step ``finding'' record: what evidence was retrieved and what was learned
        \item produce a final mechanistic summary that (i) cites retrieved evidence and (ii) aligns with CORUM annotations
    \end{enumerate}

    \item Start with templated ``finding'' statements derived directly from tool outputs (high precision), then progressively replace templates with model-generated summaries as training improves.
\end{itemize}

\subsection{Training Framework}\label{sec:training}
\begin{itemize}
    \item \textbf{Phase 0: General-knowledge tool pretraining (SFT / continued pretraining)}
    \begin{itemize}
        \item Use API-grounded tasks over MyGene.info/GO/Reactome/KEGG-style endpoints:
        gene $\rightarrow$ function, GO category, localization; pathway membership; gene synonyms/IDs; mechanistic term extraction.
        \item Objective: strengthen biological reasoning and tool literacy before CORUM/RL.
    \end{itemize}

    \item Tool-call SFT for proper schema-valid tool calling (reduce invalid actions).

    \item Evidence $\rightarrow$ interpretation SFT:
    \begin{itemize}
        \item input: tool observations + structured state
        \item target: mechanistic labels + short grounded finding statement
    \end{itemize}

    \item \textbf{Offline preference optimization with CORUM-derived scores} (no expert model):
    \begin{itemize}
        \item context $x$: current trajectory state
        \item candidates $\{y_i\}$: next-step proposals (tool call + state update + optional finding)
        \item rank candidates by deterministic reward components (tool validity, $\Delta$F1, mechanistic label match, groundedness)
        \item build $\{\text{prompt}, \text{chosen}, \text{rejected}\}$ for DPO
    \end{itemize}

    \item \textbf{Online RL for long-horizon behavior} (PPO/GRPO-style):
    \begin{itemize}
        \item full episodes with model controlling actions under compute budgets
        \item group rollouts for variance reduction; trajectory-level outcome rewards propagated to planning decisions
    \end{itemize}
\end{itemize}

\subsection{Reward Design and Decomposition}\label{sec:rewards}
\begin{itemize}
    \item Query validity and tool discipline:
    \begin{itemize}
        \item + valid schema, executes
        \item + non-empty returns when appropriate
        \item -- per tool call (efficiency) and repeated invalid loops
    \end{itemize}

    \item Complex recovery reward:
    \begin{itemize}
        \item set-based metrics: F1/Jaccard/precision@k/recall@k
        \item dense shaping: + for adding correct unseen members; -- for adding incorrect proteins
    \end{itemize}

    \item Mechanistic interpretation reward:
    \begin{itemize}
        \item compare predicted mechanistic labels to CORUM annotations
        \item groundedness: + for using retrieved terms/evidence; -- for unsupported claims
    \end{itemize}

    \item Topological/contextual rewards:
    \begin{itemize}
        \item + explore complementary neighborhoods; -- redundant revisits
    \end{itemize}
\end{itemize}

\subsection{Curriculum and Optimization Strategy}\label{sec:curriculum}
\begin{itemize}
    \item 1: intra-complex completion (short horizon, dense rewards)

    \item 2: intra-complex with decoys + noisy graphs

    \item 3: inter-complex reasoning (harder; encourages broader context / diffusion)
\end{itemize}

\subsection{Evaluation and Benchmarks}\label{sec:eval}
\begin{itemize}
    \item Automatic:
    \begin{itemize}
        \item held-out CORUM recovery (F1/Jaccard/precision@k), mechanistic label accuracy
        \item tool efficiency: tool calls per successful recovery; latency/compute budgets
        \item robustness to graph noise / dropped edges
    \end{itemize}

    \item Human evaluation for summaries (adapt rubric used in existing agentic interpretation systems):
    \begin{itemize}
        \item accuracy, relevance, biological plausibility, hypothesis generation, integration
        \item explicit checks for provenance-grounded statements
    \end{itemize}
\end{itemize}
